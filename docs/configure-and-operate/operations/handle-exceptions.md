이 섹션에서는 Apache Ignite가 발생시킬 수 있는 기본 예외와 이를 처리하는 기본 방법을 설명합니다.

## 스택 트레이스 정보 찾기 {#finding-stack-trace-information}

예외가 발생하면 Apache Ignite는 해당 예외의 UUID를 제공하지만 전체 스택 트레이스는 제공하지 않습니다. 전체 스택 트레이스를 확인하려면 클러스터 로그를 확인하세요.

## 인증 예외 {#authentication-exceptions}

| 예외 코드 | 설명 | 권장 조치 |
|---|---|---|
| `IGN-AUTHENTICATION-1` | 지원하지 않는 인증 유형으로 인한 인증 오류입니다. | 클라이언트 구성을 확인하고 지원되는 구성 유형을 사용하세요. |
| `IGN-AUTHENTICATION-2` | 잘못된 자격 증명으로 인한 인증 오류입니다. | 사용자 자격 증명을 확인하고 수정하세요. |
| `IGN-AUTHENTICATION-3` | 기본 인증 공급자를 찾을 수 없습니다. | 기본 인증 공급자를 구성하세요. |

## 클라이언트 예외 {#client-exceptions}

| 예외 코드 | 설명 | 권장 조치 |
|---|---|---|
| `IGN-CLIENT-1` | 클라이언트 연결에 실패했습니다. 이 오류는 보통 잘못된 주소나 연결 타임아웃 때문에 발생합니다. | 예외 메시지를 확인하세요. 타임아웃이 발생했다면 클라이언트와 서버 구성의 타임아웃 값이 장시간 실행되는 요청을 허용하는지 확인하세요. |
| `IGN-CLIENT-2` | 연결 프로토콜에 문제가 발생했습니다. 이 오류는 호환되지 않는 클라이언트 버전을 사용했거나 데이터가 손상되어 발생할 수 있습니다. | 예외 메시지를 확인하세요. 데이터가 손상됐다면 데이터를 복구하세요. 클라이언트 버전이 호환되지 않는다면 올바른 클라이언트 버전을 사용하세요. |
| `IGN-CLIENT-3` | 프로토콜 버전이 특정 기능을 지원하지 않습니다. 보통 클라이언트와 서버 간 버전 불일치로 발생합니다. | 클라이언트나 서버를 최신 버전으로 업데이트하거나 호환되지 않는 기능을 사용하지 마세요. |
| `IGN-CLIENT-4` | ID로 테이블을 찾지 못했습니다. | 테이블 ID가 올바른지 확인하세요. |
| `IGN-CLIENT-5` | 클라이언트 구성에 오류가 발생했습니다. | 자세한 내용은 예외 메시지를 확인하고 클라이언트 구성을 수정하세요. |
| `IGN-CLIENT-6` | 클러스터 ID가 일치하지 않습니다. 이 오류는 클라이언트가 여러 개의 서로 다른 클러스터에 연결할 때 발생할 수 있습니다. | 모든 엔드포인트가 같은 클러스터에 속하는지 클라이언트 구성을 확인하세요. |
| `IGN-CLIENT-7` | 클라이언트 SSL 구성이 올바르지 않습니다. | 자세한 내용은 예외 메시지를 확인하세요. SSL 구성이 서버 측 구성과 일치하는지 확인하세요. |
| `IGN-CLIENT-8` | 클라이언트 핸드셰이크 메시지 오류입니다. 보통 클라이언트가 잘못된 엔드포인트(예: REST)에 연결을 시도하거나, Apache Ignite가 아닌 프로세스가 Apache Ignite 엔드포인트에 연결을 시도할 때 발생합니다. | 자세한 내용은 예외 메시지를 확인하세요. 문제가 클라이언트에 있다면 클라이언트 구성을 확인하고 수정하세요. 서드파티 클라이언트가 연결을 시도하고 있다면 어떤 클라이언트인지 확인하고 구성을 수정하세요. |
| `IGN-CLIENT-9` | 서버가 클라이언트에 보낸 요청이 실패했습니다. | 네트워크 연결 상태, 클라이언트 가용성, 서버 로그에서 오류를 확인하세요. |

| `IGN-TX-9` | 트랜잭션 상태 스토리지 리밸런싱 오류입니다. | 리밸런싱 오류의 원인을 해결하고, 디스크 오류가 없는지 확인하세요. |
| `IGN-TX-10` | 테이블에서 사용 가능한 데이터보다 오래된 타임스탬프로 읽기 전용 트랜잭션을 생성하지 못했습니다. | 더 최신 읽기 타임스탬프로 트랜잭션을 다시 시도하세요. 데이터 가용 시간을 늘리세요. |
| `IGN-TX-11` | 호환되지 않는 스키마 변경으로 인한 실패입니다. | 트랜잭션이 호환되지 않는 스키마 내의 데이터에 접근을 시도했습니다. 트랜잭션을 다시 시도하세요. |
| `IGN-TX-12` | 프라이머리 복제본 만료로 인한 실패입니다. | 트랜잭션을 다시 시도하세요. 프라이머리 복제본 만료를 유발할 수 있는 노드 장애, 네트워크 문제, JVM/GC 일시 정지가 없는지 확인하세요. |
| `IGN-TX-13` | 트랜잭션이 이미 종료됐습니다. | 종료(커밋 또는 중단)된 트랜잭션 내에서 어떤 작업을 수행하든 발생할 수 있습니다. 별도 조치가 필요하지 않습니다. |
| `IGN-TX-14` | 완료된 트랜잭션의 오래된 작업으로 인한 실패입니다. | 트랜잭션의 오래된 작업이 감지될 때 발생하는 오류입니다. 필요하면 트랜잭션을 다시 실행해 보세요. |
| `IGN-TX-15` | 트랜잭션의 읽기 타임스탬프에 해당하는 데이터가 이미 삭제된 노드에서 읽기 전용 트랜잭션의 작업을 실행하려 할 때 발생하는 오류입니다. | 더 최신 읽기 타임스탬프로 트랜잭션을 다시 시도하세요. 데이터 가용 시간을 늘리세요. |
| `IGN-TX-16` | 트랜잭션이 타임아웃으로 이미 종료되어 작업이 실패했습니다. | 트랜잭션 타임아웃 설정을 확인하고 트랜잭션이 만료되기 전에 작업이 실행되는지 확인하세요. |

## 알 수 없는 오류 코드 {#unknown-error-codes}

| 예외 코드 | 설명 | 권장 조치 |
|---|---|---|
| `UNKNOWN-UNKNOWN{groupCode}-{errorCode}` | 클라이언트가 인식하지 못하는 오류 코드를 받았습니다. 이 오류는 클라이언트가 모르는 오류 코드를 도입한 최신 버전의 서버에 이전 버전의 클라이언트가 연결할 때 발생합니다. | 서버와 호환되는 버전으로 클라이언트를 업데이트해 올바른 오류 정보를 받으세요. 전체 오류 컨텍스트는 서버 로그를 확인하세요. |
