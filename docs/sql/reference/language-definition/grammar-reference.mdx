## column_definition

<RailroadDiagram>{`
Diagram(

<RailroadDiagram>{`

Start({type:'complex'}),

Optional(Sequence(Terminal('DEFAULT'), Choice(1,NonTerminal('identifier'),
NonTerminal('literal_value'),
Sequence(Terminal('CURRENT_TIMESTAMP'), Sequence(Terminal('+'),Terminal('INTERVAL'), NonTerminal('interval'))
)
),)),
Optional(Terminal('PRIMARY KEY')),
))
`}</RailroadDiagram>

키워드와 매개변수:

* `column_name` - 컬럼 이름입니다.
* `DATA TYPE` - 컬럼에 허용되는 [데이터 타입](/sql/reference/data-types-and-functions/data-types)입니다.
* `identifier` - 행의 임의 식별자입니다. [rand_uuid](/sql/reference/data-types-and-functions/operators-and-functions#rand_uuid) 함수로 생성할 수 있습니다.
* `literal_value` - 기본값으로 할당할 값입니다.
* `CURRENT_TIMESTAMP` - 현재 시각을 반환하는 함수입니다. `TIMESTAMP WITH LOCAL TIME ZONE` 컬럼에만 사용할 수 있습니다.
* `interval` - 타임스탬프의 오프셋으로 사용할 시간 간격입니다. `CURRENT_TIMESTAMP`를 사용하면 필수입니다. 간격은 다음 단위로 지정할 수 있습니다:
  - `SECONDS`
  - `MINUTES`
  - `HOURS`
  - `DAYS`
  - `MONTHS`

  간격이 `0`이면 현재 시각을 사용합니다.

## parameter

<RailroadDiagram>{`
Diagram(
NonTerminal('parameter_name'),
Terminal('='),
NonTerminal('parameter_value'))
`}</RailroadDiagram>

매개변수:

* `parameter_name` - 매개변수의 이름입니다.
* `parameter_value` - 매개변수의 값입니다.

매개변수를 지정할 때는 리터럴 값이나 식별자로 제공할 수 있습니다. 예를 들면 다음과 같습니다:

```
CREATE ZONE test_zone;
CREATE TABLE test_table (id INT PRIMARY KEY, val INT) WITH PRIMARY_ZONE=test_zone;
```

이 경우 `test_zone`은 식별자이며 식별자로 사용됩니다. 이렇게 사용하면 매개변수는 대소문자를 구분하지 않습니다.

```
CREATE ZONE "test_zone";
CREATE TABLE test_table (id INT PRIMARY KEY, val INT) WITH PRIMARY_ZONE='test_zone';
```

이 경우 `test_zone`은 리터럴 값으로 생성되어 리터럴로 사용됩니다. 이렇게 사용하면 매개변수는 대소문자를 구분합니다.

```
CREATE ZONE test_zone;
CREATE TABLE test_table (id INT PRIMARY KEY, val INT) WITH PRIMARY_ZONE=`TEST_ZONE`;
```

이 경우 `test_zone`은 식별자로 생성되며 대소문자를 구분하지 않습니다. 따라서 `TEST_ZONE`을 리터럴로 사용해도 식별자와 일치합니다.


다음에서 참조됨:

* [CREATE ZONE](/sql/reference/language-definition/distribution-zones#create-zone)
* [ALTER ZONE](/sql/reference/language-definition/distribution-zones#alter-zone)
