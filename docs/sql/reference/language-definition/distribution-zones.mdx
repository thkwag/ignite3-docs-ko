# 분산 영역

이 섹션에서는 Apache Ignite 3의 분산 영역(distribution zone)을 설명합니다. Apache Ignite 3에서는 파티션을 노드에 배치하는 방식을 세밀하게 조정해 성능과 안정성을 높일 수 있습니다.

## CREATE ZONE

새 분산 영역을 생성합니다.

:::note
이 작업은 [Java API](/develop/work-with-data/java-to-tables)로도 수행할 수 있습니다.
:::

### 키워드와 매개변수 {#keywords-and-parameters}

- `IF NOT EXISTS` - 같은 이름의 다른 영역이 없을 때만 영역을 생성합니다.
- `qualified_zone_name` - 분산 영역의 이름입니다. 대소문자를 구분하는 문자열이나 대소문자를 구분하지 않는 식별자로 지정할 수 있습니다. 테이블을 생성하는 시점에 존재하지 않아도 되며, 데이터를 쓰기 전에 생성할 수 있습니다.
- `STORAGE PROFILES` - 필수입니다. 사용할 스토리지 엔진의 프로파일을 쉼표로 구분한 목록입니다.
- `PARTITIONS` - 데이터를 나눌 파티션 수입니다. 이후 파티션은 저장을 위해 노드 간에 분할됩니다.
- `REPLICAS` - 각 파티션의 복제본 수입니다.
- `NODES FILTER` - 노드 속성을 기준으로 분산 영역에 데이터를 저장할 수 있는 노드를 지정합니다. 노드 속성은 CLI로 구성할 수 있습니다. 필터는 JSONPath 규칙을 사용합니다. 해당 속성이 없으면 모든 부정 비교가 유효한 것으로 처리됩니다. 예를 들어, `$[?(@.storage != 'SSD']}`는 `storage` 속성이 지정되지 않은 노드도 포함합니다.
- `AUTO SCALE UP` - 분산 영역의 자동 확장을 설정합니다. 사용 가능한 값:
  - `OFF` - 분산 영역 자동 조정을 비활성화합니다.
  - `integer` - 새 노드가 참여한 시점부터 분산 영역 조정이 시작되기까지의 지연 시간(초)입니다. 가능한 값은 0부터 2147483647까지입니다. 기본값: 5.
- `AUTO SCALE DOWN` - 분산 영역의 자동 축소를 설정합니다. 사용 가능한 값:
  - `OFF` - 분산 영역 자동 조정을 비활성화합니다. 기본값입니다.
  - `integer` - 새 노드가 참여한 시점부터 분산 영역 조정이 시작되기까지의 지연 시간(초)입니다. 가능한 값은 0부터 2147483647까지입니다.
- `CONSISTENCY MODE` - 영역이 파티션 과반수 상실을 처리하는 방식입니다. `STRONG CONSISTENCY`로 설정하면 과반수가 복구될 때까지(일반적으로 노드가 클러스터를 이탈했다가 다시 참여하는 것을 의미합니다) 데이터를 사용할 수 없습니다. `HIGH AVAILABILITY`는 남은 노드에서 데이터를 읽고 쓰며, 데이터 손실 가능성을 감수합니다. 기본값: `STRONG_CONSISTENCY`.
- `QUORUM SIZE` - 복제 그룹의 합의에서 과반수를 이루는 노드 수입니다. 정족수 크기는 복제본 수에 따라 결정됩니다.

  _최솟값_: 복제본이 하나뿐이면 `1`, 둘 이상이면 `2`입니다.
  _최댓값_: 전체 복제본 수의 절반을 올림한 값입니다.

  _기본값_: 복제본이 5개 이상이면 `3`입니다. 그 외에는 2와 전체 복제본 수 중 더 작은 값입니다.

### 예시 {#examples}

_대소문자를 구분하지 않는_ 식별자로 지정된 `exampleZone` 분산 영역을 생성합니다:

```sql
CREATE ZONE IF NOT EXISTS exampleZone STORAGE PROFILES['default'];
```

_대소문자를 구분하는_ 문자열로 지정된 `"myExampleZone"` 분산 영역을 생성합니다:

```sql
CREATE ZONE IF NOT EXISTS "myExampleZone" STORAGE PROFILES['default'];
```

클러스터 토폴로지가 변경된 후 노드를 추가하기까지 300초를 기다리도록 `exampleZone` 분산 영역을 생성합니다:

```sql
CREATE ZONE IF NOT EXISTS exampleZone (AUTO SCALE UP 300) STORAGE PROFILES['default'];
```

클러스터 토폴로지가 변경되면 유휴 노드를 축소하기 전 600초를 기다리도록 `exampleZone` 분산 영역을 생성합니다:

```sql
CREATE ZONE IF NOT EXISTS exampleZone (AUTO SCALE DOWN 600) STORAGE PROFILES['default'];
```

`SSD` 속성이 있는 노드에만 데이터가 저장되는 `exampleZone` 분산 영역을 생성합니다:

```sql
CREATE ZONE IF NOT EXISTS exampleZone (NODES FILTER '$[?(@.storage == "SSD")]') STORAGE PROFILES['default'];
```

:::note
위 예시는 [노드 필터링](/sql/reference/language-definition/distribution-zones) 예시에서와 같이 속성이 설정된 노드가 있다고 가정합니다.
:::

일관성 모드를 `HIGH_AVAILABILITY`로 설정한 `exampleZone` 분산 영역을 생성합니다:

```sql
CREATE ZONE IF NOT EXISTS exampleZone (REPLICAS 5, CONSISTENCY MODE 'HIGH_AVAILABILITY') STORAGE PROFILES['default'];
```

## ALTER ZONE

분산 영역을 수정합니다.

### ALTER ZONE RENAME TO new_qualified_zone_name

<RailroadDiagram>{`
Diagram(
Terminal('ALTER ZONE'),
Optional(Terminal('IF EXISTS')),
NonTerminal('qualified_zone_name'),
Terminal('RENAME TO'),
NonTerminal('new_qualified_zone_name'),
)
`}</RailroadDiagram>

#### 키워드와 매개변수 {#keywords-and-parameters-1}

- `IF EXISTS` - 지정한 이름의 영역이 없어도 오류를 발생시키지 않습니다.
- `qualified_zone_name` - 분산 영역의 현재 이름입니다.
- `RENAME TO` - 선택한 영역의 이름을 새 이름으로 바꿉니다.
- `new_qualified_zone_name` - `RENAME`으로 지정하는 분산 영역의 새 이름입니다.

#### 예시 {#examples-1}

`exampleZone`의 이름을 `renamedZone`으로 바꿉니다:

```sql
ALTER ZONE IF EXISTS exampleZone RENAME TO renamedZone;
```

- `IF EXISTS` - 지정한 이름의 영역이 없어도 오류를 발생시키지 않습니다.
- `qualified_zone_name` - 분산 영역의 이름입니다.
- `SET` - 다음 매개변수 중 일부 또는 전체에 값을 지정합니다:
  - `PARTITIONS` - 파티션 수입니다.
  - `REPLICAS` - 각 파티션의 복제본 수입니다.
  - `NODES FILTER` - 노드 속성을 기준으로 분산 영역에 데이터를 저장할 수 있는 노드를 지정합니다.
  - `AUTO SCALE` - 분산 영역의 자동 조정을 설정합니다. 사용 가능한 값:
    - `OFF` - 분산 영역 자동 조정을 비활성화합니다. 기본값입니다.
    - `integer` - 새 노드가 참여한 시점부터 분산 영역 조정이 시작되기까지의 지연 시간(초)입니다. 가능한 값은 0부터 2147483647까지입니다.
  - `AUTO SCALE UP` - 분산 영역의 자동 확장을 설정합니다. 사용 가능한 값:
    - `OFF` - 분산 영역 자동 조정을 비활성화합니다. 기본값입니다.
    - `integer` - 새 노드가 참여한 시점부터 분산 영역 조정이 시작되기까지의 지연 시간(초)입니다. 가능한 값은 0부터 2147483647까지입니다.
  - `AUTO SCALE DOWN` - 분산 영역의 자동 축소를 설정합니다. 사용 가능한 값:
    - `OFF` - 분산 영역 자동 조정을 비활성화합니다. 기본값입니다.
  - `QUORUM SIZE` - 복제 그룹의 합의에서 과반수를 이루는 노드 수입니다. 정족수 크기는 복제본 수에 따라 결정됩니다.

    _최솟값_: 복제본이 하나뿐이면 `1`, 둘 이상이면 `2`입니다.
    _최댓값_: 전체 복제본 수의 절반을 올림한 값입니다.

    _기본값_: 복제본이 5개 이상이면 `3`입니다. 그 외에는 2와 전체 복제본 수 중 더 작은 값입니다.

#### 예시 {#examples-2}

데이터 복제본 수를 10으로 설정합니다:

```sql
ALTER ZONE IF EXISTS exampleZone SET REPLICAS 10;
```

특정 지역과 일치하도록 데이터 노드 필터를 설정하고, 데이터 복제본 수를 5로, 정족수 크기를 3으로 설정합니다:

```sql
ALTER ZONE IF EXISTS exampleZone SET (REPLICAS 5, QUORUM SIZE 3, NODES FILTER '$[?(@.region == "US")]');
```

- `IF EXISTS` - 지정한 이름의 영역이 없어도 오류를 발생시키지 않습니다.
- `qualified_zone_name` - 분산 영역의 이름입니다.

### 예시 {#examples-3}

```sql
DROP ZONE IF EXISTS exampleZone;
```
