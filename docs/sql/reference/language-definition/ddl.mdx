이 섹션에서는 Apache Ignite 3가 지원하는 모든 DDL 명령어를 설명합니다.

## CREATE TABLE

새 테이블을 생성합니다.

:::note
이 작업은 [Java API](../../../develop/work-with-data/java-to-tables)로도 수행할 수 있습니다.
:::

`}</RailroadDiagram>

<RailroadDiagram>{`
Diagram(
Start({type:'complex'}),
Optional(
Sequence(

)
),
Optional(
Sequence(
Terminal('STORAGE PROFILE'),
NonTerminal('profile_name')
)
)
)
`}</RailroadDiagram>

키워드와 매개변수:

* `IF NOT EXISTS` - 같은 이름의 테이블이 없을 때만 테이블을 생성합니다.
* `COLOCATE BY` - 콜로케이션 키입니다. 복합 키를 사용할 수 있으며, 기본 키는 반드시 콜로케이션 키를 포함해야 합니다.
* `ZONE` - [분산 영역(distribution zone)](/sql/reference/language-definition/distribution-zones)을 설정합니다. 대소문자를 구분하는 문자열이나 대소문자를 구분하지 않는 식별자로 지정할 수 있습니다. 테이블을 생성하는 시점에 존재하지 않아도 되며, 데이터를 쓰기 전에 생성할 수 있습니다.
* `STORAGE PROFILE` - 테이블을 저장할 [스토리지 프로파일](../../../configure-and-operate/reference/node-configuration)을 설정합니다. 대소문자를 구분하는 문자열로 지정해야 합니다.

예시:

Person 테이블을 생성합니다:

```sql
CREATE TABLE IF NOT EXISTS Person (
  id int primary key,
  city_id int,
  name varchar,
  age int,
  company varchar
)
```

`MYZONE` 분산 영역을 사용하는 Person 테이블을 생성합니다:

```sql
CREATE TABLE IF NOT EXISTS Person (
  id int primary key,
  city_id int,
  name varchar,
  age int,
  company varchar
) ZONE MYZONE
```

분산 영역에 지정된 스토리지 프로파일과 관계없이 `default` 스토리지 프로파일을 사용하는 Person 테이블을 생성합니다:

```sql
CREATE TABLE IF NOT EXISTS Person (
  id int primary key,
  city_id int,
  name varchar,
  age int,
  company varchar
) PRIMARY ZONE MYZONE STORAGE PROFILE 'default'
```

`city_id` 컬럼의 기본값이 1인 Person 테이블을 생성합니다:

```sql
CREATE TABLE IF NOT EXISTS Person (
  id int primary key,
  city_id int default 1,
  name varchar,
  age int,
  company varchar
)
```

키워드와 매개변수:

* `IF EXISTS` - 지정한 이름의 테이블이 없어도 오류를 발생시키지 않습니다.

예시:

테이블에 컬럼을 추가합니다:

```sql
ALTER TABLE Person ADD COLUMN city varchar;
```

테이블이 있을 때만 컬럼을 추가합니다:

```sql
ALTER TABLE IF EXISTS Person ADD number bigint;
```

테이블에 여러 컬럼을 한 번에 추가합니다:

```sql
ALTER TABLE Person ADD COLUMN (code varchar, gdp double);
```

### ALTER TABLE IF EXISTS table DROP COLUMN (column1, column2 int)

기존 테이블에서 컬럼을 삭제합니다. 컬럼을 삭제하면 쿼리에서 더 이상 접근할 수 없습니다. 이 명령어에는 다음과 같은 제약이 있습니다.

- 컬럼에 인덱스가 있다면 'DROP INDEX' 명령어로 미리 수동으로 인덱스를 삭제해야 합니다.
- 클러스터에 저장된 값 전체를 나타내는 컬럼은 삭제할 수 없습니다. 이 제약은 프리미티브 값에 해당합니다.

End({type:'complex'})
)
`}</RailroadDiagram>

<RailroadDiagram>{`

키워드와 매개변수:

* `IF EXISTS` - 지정한 이름의 테이블이 없어도 오류를 발생시키지 않습니다.

예시:

테이블에서 컬럼을 삭제합니다:

```sql
ALTER TABLE Person DROP COLUMN city;
```

테이블이 있을 때만 컬럼을 삭제합니다:

```sql
ALTER TABLE IF EXISTS Person DROP COLUMN number;
```

테이블에서 여러 컬럼을 한 번에 삭제합니다:

```sql
ALTER TABLE Person DROP COLUMN (code, gdp);
```

* `data_type` - 유효한 [데이터 타입](/sql/reference/data-types-and-functions/data-types)입니다.

```sql
ALTER TABLE Person ALTER COLUMN city SET DATA TYPE varchar;
```

#### 지원되는 전환 {#supported-transitions}

모든 데이터 타입 전환이 지원되는 것은 아닙니다. 제약 사항은 다음과 같습니다.

* `FLOAT`는 `DOUBLE`로 전환할 수 있습니다
* `INT8`, `INT16`, `INT32`는 `INT64`로 전환할 수 있습니다
* `TYPE SCALE` 변경은 금지됩니다
* PK가 아닌 DECIMAL 컬럼은 `TYPE PRECISION` 증가가 허용됩니다
* PK가 아닌 STRING·BYTE_ARRAY 컬럼은 `TYPE LENGTH` 증가가 허용됩니다

그 외의 전환은 지원되지 않습니다.

예시:

ID의 가능한 범위를 BIGINT 범위로 변경합니다:

```sql
ALTER TABLE Person ALTER COLUMN age SET DATA TYPE BIGINT
```

컬럼 텍스트의 길이를 11로 설정합니다:

```sql
ALTER TABLE Person ALTER COLUMN Name SET DATA TYPE varchar(11)
```

## DROP TABLE

`DROP TABLE` 명령어는 기존 테이블을 삭제합니다. 테이블은 삭제 대상으로 표시되며 [하한 워터마크](../../../understand/core-concepts/data-partitioning#version-storage) 지점에 도달한 후 가비지 컬렉션으로 제거됩니다. 데이터가 제거되기 전까지는, 테이블이 삭제 대상으로 표시되기 이전 시점을 기준으로 조회하는 [읽기 전용 트랜잭션](../../../develop/work-with-data/transactions#read-only-transactions)에서 이 데이터를 계속 사용할 수 있습니다.

:::note
이 작업은 [Java API](../../../develop/work-with-data/java-to-tables)로도 수행할 수 있습니다.
:::

<RailroadDiagram>{`
Diagram(
Terminal('DROP TABLE'),
Optional(Terminal('IF EXISTS')),
NonTerminal('qualified_table_name', {href:'./grammar-reference#qualified_table_name'})
)
`}</RailroadDiagram>

키워드와 매개변수:

* `IF EXISTS` - 같은 이름의 테이블이 없어도 오류를 발생시키지 않습니다.

예시:

Person 테이블이 있으면 삭제합니다:

```sql
DROP TABLE IF EXISTS "Person";
```

## CREATE INDEX

새 인덱스를 생성합니다.

:::note
이 작업은 [Java API](../../../develop/work-with-data/java-to-tables)로도 수행할 수 있습니다.
:::

새 인덱스를 생성하면, 인덱스 생성 이전에 시작된 모든 트랜잭션이 완료된 후에야 인덱스 빌드가 시작됩니다. 클러스터의 논리 토폴로지에 멈춰 있는("hung") 트랜잭션이 있으면 인덱스 빌드가 시작되지 않습니다.

인덱스 상태는 상태 사유 설명(예: PENDING - "Waiting for transaction ABC to complete")과 함께 시스템 뷰에 반영됩니다.

:::note
인덱스에는 같은 컬럼을 두 번 이상 포함할 수 없습니다.
:::

<RailroadDiagram>{`
Diagram(
  Terminal('CREATE INDEX'),
  Optional(Terminal('IF NOT EXISTS')),
  NonTerminal('name'),
  Terminal('ON'),
  NonTerminal('qualified_table_name', {href:'./grammar-reference#qualified_table_name'}),
  End({type:'complex'})
)
`}</RailroadDiagram>

<RailroadDiagram>{`
Diagram(
Start({type:'complex'}),
Sequence(
Choice (0,
Sequence(
Terminal('USING'),
Choice (0,
Sequence('TREE',NonTerminal ('sorted_column_list', {href:'./grammar-reference#sorted_column_list'})),
Sequence('HASH',NonTerminal ('column_list', {href:'./grammar-reference#column_list'}))
),
),
NonTerminal ('sorted_column_list', {href:'./grammar-reference#sorted_column_list'})
)),
End({type:'simple'})
)
`}</RailroadDiagram>

키워드와 매개변수:

* `IF NOT EXISTS` - 같은 이름의 인덱스가 없을 때만 인덱스를 생성합니다.
* `name` - 인덱스의 이름입니다.
* `ON` - 지정한 테이블에 인덱스를 생성합니다.
* `USING TREE` - 지정하면 트리 인덱스를 생성합니다.
* `USING HASH` - 지정하면 해시 인덱스를 생성합니다.

예시:

Person 테이블에 `department_name_idx` 인덱스를 생성합니다:

```sql
CREATE INDEX IF NOT EXISTS department_name_idx ON Person (department_id DESC, name ASC);
```

Person 테이블에 해시 인덱스 `department_name_idx`를 생성합니다:

```sql
CREATE INDEX name_surname_idx ON Person USING HASH (name, surname);
```

Person 테이블에 트리 인덱스 `department_city_idx`를 생성합니다:

```sql
CREATE INDEX department_city_idx ON Person USING TREE (department_id ASC, city_id DESC);
```

## DROP INDEX

인덱스를 삭제합니다.

:::note
이 작업은 [Java API](../../../develop/work-with-data/java-to-tables)로도 수행할 수 있습니다.
:::

인덱스를 삭제하면, `DROP INDEX` 명령 이전에 시작된 모든 트랜잭션이 완료될 때까지 인덱스는 STOPPING 상태를 유지합니다(삭제 대상 인덱스가 속한 테이블에 영향을 주지 않는 트랜잭션도 포함합니다).
위 트랜잭션이 모두 완료된 후에도, 삭제된 인덱스가 차지하던 공간은 해당 파티션의 LWM이 인덱스 삭제가 시작된 시점보다 커져야만 회수됩니다.
인덱스 상태는 상태 사유 설명(예: PENDING - "Waiting for transaction ABC to complete")과 함께 시스템 뷰에 반영됩니다.

<RailroadDiagram>{`
Diagram(
Terminal('DROP INDEX'),
Optional(Terminal('IF EXISTS')),
NonTerminal('index_name')
)
`}</RailroadDiagram>

키워드와 매개변수:

* `index_name` - 인덱스의 이름입니다.
* `IF EXISTS` - 지정한 이름의 인덱스가 없어도 오류를 발생시키지 않습니다.

예시:

인덱스가 있으면 삭제합니다:

```sql
DROP INDEX IF EXISTS department_name_idx;
```
