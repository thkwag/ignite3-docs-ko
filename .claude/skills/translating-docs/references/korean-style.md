# 한국어 번역 문체 규칙

번역·검수 시 이 문서의 패턴을 기준으로 문장 품질을 판단한다. 기계 검사가 가능한 패턴은
`GLOSSARY.md`의 "금지 표현" 표에 등록되어 `check-glossary.mjs`가 잡아내고,
이 문서는 그 배경 원칙과 감사가 못 잡는 판단 기준을 다룬다.

## 핵심 원칙

영어 문장 구조를 한국어 단어로 치환하지 않는다. 의미를 이해한 뒤 한국어로 다시 쓴다.
주어(you/we/it)는 생략하고, 피동보다 능동, 미래 단정("~할 것입니다")보다 현재형 단언("~합니다")을 쓴다.

## 번역투 패턴과 처방

심각도: **S1** = 한 번만 나와도 고친다. **S2** = 반복되면(대략 3회+) 고친다.

| 패턴 | 심각도 | 처방 |
| ---- | ---- | ---- |
| "~에 대해(서)/~에 대한" 남발 | S2 | 목적격 조사로 직결: "X에 대해 설명" → "X를 설명" |
| "~를 통해/통하여" 남발 | S2 | "~로", "~해서", "~함으로써"로 분산 |
| "~에 있어(서)" | S1 | "~에서", "~할 때" |
| "~와 관련하여/관련된" 남발 | S2 | "~에", "~의" |
| 명사화 과잉 "~에 기반하여/바탕으로" | S2 | 동사로 환원: "성능의 향상" → "성능을 높이려면" |
| "가지고 있다" (have 직역) | S1 | "~가 있다", "~를 제공한다" |
| 이중 피동 "되어진다/되어집니다/보여진다" | S1 | 능동 또는 단일 피동: "판단되어진다" → "판단된다" |
| "~에 의해" 피동 남발 | S2 | 행위자를 주어로: "노드에 의해 처리" → "노드가 처리" |
| "~하는 것을 허용한다/가능하게 한다" | S1 | "~할 수 있다" |
| "~을 위해" 목적절 남발 | S2 | "~하려면", "~용" |
| 대명사 직역 "그/그것/그들" 반복 | S1 | 생략(한국어는 무주어가 자연스럽다) 또는 명사 반복 |
| 복수 접미사 "-들" 남발 | S2 | 삭제: "노드들이" → "노드가" |
| 명사 앞 3어절+ 관형절 좌향 수식 | S2 | 문장 분리 또는 어순 재구성 |
| 이중 조사 "~에서의/~으로의/~에의" | S2 | 절·구로 풀어쓰기 |
| "~할 것입니다" (will 직역) | S2 | 사실 서술은 현재형 "~합니다" |
| "~할 수 있습니다" (can) 남발 | S2 | 단언 가능하면 "~합니다". 기능 설명 자체는 허용 |
| 음차 남발: 디폴트, 레버리지, 이슈(문제 의미) | S1 | 기본값, 활용, 문제 |
| 어색한 한자어: 함의, 동치, 정련, 가역 | S2 | 뒷받침, 일치, 정제, 되돌릴 수 있는 |
| 문두 접속사 "또한·따라서·즉" 문단마다 반복 | S1 | 대량 제거, 문장 흐름으로 연결 |
| "본질적으로/핵심적으로" | S1 | 삭제 또는 구체 서술 |
| hype 어휘 남발 (강력한, 획기적, 완벽한) | S2 | 구체 수치·사실로 |
| 한글+괄호 영어 매번 병기 | S2 | 문서당 첫 등장만 병기, 이후 한국어만 |

## 문체

- 본문은 합니다체, 독자 지시는 "~하세요".
- 원문이 명령문이면 "~하세요/합니다"로, 설명문이면 평서형으로 옮긴다. 원문 어조(주의·경고 수위)를 유지한다.
- 조사(은/는·이/가·을/를·과/와)는 앞 단어의 받침에 맞춘다. 영어 원문 유지 단어 뒤 조사는 실제 발음 기준(예: "Ignite는", "RAFT 그룹은").
- 문장이 원문보다 길어지면 둘로 나눈다. 영어 관계절을 그대로 긴 관형절로 옮기지 않는다.

## 과잉 교정 금지

- 정착 외래어는 유지: 어댑터, 콜백, 매핑, 스냅샷, 렌더링, 파싱, 핸들러, 메타데이터, 클러스터, 노드, 파티션, 스트리밍
- 고유명사·수치·날짜·단위·코드 식별자는 한 글자도 바꾸지 않는다
- 자연스럽게 하겠다며 원문에 없는 비유·수사·내용을 추가하지 않는다(원문 충실도 우선)
